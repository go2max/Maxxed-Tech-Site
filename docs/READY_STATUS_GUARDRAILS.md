# Ready Status Guardrails

Use these labels consistently on the website.

- Ready candidate: built or merged enough to market, final acceptance may remain.
- Android release candidate: Android build and test evidence exists, store path may remain.
- Public site: live or canonical public website content.
- Internal testing: tester recruitment only.
- Beta candidate: beta interest and limited testing only.
- Functional debug: implementation exists but production gaps remain.
- Prototype: concept/scaffold/testing only.
- Hold / verify: blocked by an explicit verification item.
- Requestable: demand capture only.
- Pipeline: catalog visibility only.
