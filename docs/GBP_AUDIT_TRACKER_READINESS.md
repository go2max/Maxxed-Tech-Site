# GBP Audit Tracker Readiness

## Build target

GBP Audit Tracker is now a first-class browser tool in the generated product catalog. The source implementation lives under `tools/gbp-audit-tracker/`, the product metadata lives in `content/tool-products.mjs`, and the deployable browser app is served from `public/tools/gbp-audit-tracker/app/`.

It is intentionally manual-first: no Google APIs, no scraping, no backend, no accounts, and no payment logic.

## Public routes

After the normal site build:

```text
/tools/gbp-audit-tracker/      Generated first-class product landing page
/tools/gbp-audit-tracker/app/  Browser-local audit app
```

The landing page is generated by `scripts/build.mjs`, included in the sitemap, and linked from the public catalog. The app route is copied from `public/` into the generated site.

## Validation commands

```bash
npm run gbp:check
```

Manual source smoke test:

```bash
python3 -m http.server 4173
# open http://localhost:4173/tools/gbp-audit-tracker/
```

Manual deploy-app smoke test before build:

```bash
python3 -m http.server 4173
# open http://localhost:4173/public/tools/gbp-audit-tracker/app/
```

After `npm run build`, smoke the generated artifact:

```bash
python3 -m http.server 4173 --directory site
# product page: http://localhost:4173/tools/gbp-audit-tracker/
# app page:     http://localhost:4173/tools/gbp-audit-tracker/app/
```

## Manual smoke checklist

1. Open `/apps/` and confirm GBP Audit Tracker appears under Tools / Business / Content searches.
2. Open `/tools/gbp-audit-tracker/` and confirm the generated product page loads.
3. Click **Open tool** and confirm `/tools/gbp-audit-tracker/app/` loads.
4. Click **Load sample audit**.
5. Confirm score and recommendations populate.
6. Toggle several checklist items and confirm score changes.
7. Change a recommendation status to In Progress or Done.
8. Save the audit locally.
9. Load the saved audit.
10. Delete the saved audit.
11. Copy the report.
12. Download the markdown report.
13. Check mobile width around 390px and confirm the form remains usable.

## Out of scope for v1

- Google Business Profile API integration.
- Scraping Google search, maps, profile pages, or competitor data.
- Rank tracking.
- Multi-user login.
- Payments.
- PDF generation.
- Backend database.

## Known limitations

- localStorage is device/browser-specific and can be cleared by the browser.
- Scores are checklist-based and should be treated as operational guidance, not ranking prediction.
- Manual entry is required; this avoids API and scraping dependencies for v1.
