# WordPress Review Checklist

Use this while reviewing local WordPress previews.

## Pages

- Homepage
- App landing page preview
- App catalog preview
- Beta funnel preview
- Support page preview
- Policy page preview
- 404 page

## Checks

- Page title is clear.
- App identity is clear.
- Support and policy links are reachable.
- Mobile layout remains readable.
- Customer-facing copy does not expose implementation details.
- Media placeholders are marked clearly.
- Preview content does not add unsupported launch claims.

## Evidence

Capture notes in the product test template or launch checklist.
