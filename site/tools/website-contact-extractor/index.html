MVP workflow

What the first version should do

The first release stays deliberately bounded: one submitted website, a small number of likely contact pages, visible source URLs, and a user review step before anything is exported.

01

Normalize the submitted URL

Accept a website URL, resolve the domain, keep the canonical source, and show clear errors for invalid or blocked sites.

02

Extract public contact candidates

Collect emails, phone numbers, contact-page links, booking links, social profiles, address candidates, support links, privacy links, and terms links.

03

Review and correct before export

Let the user remove false positives, add notes, and keep every extracted value tied back to the page where it was found.

04

Export clean records

Export a copy-friendly contact card, CSV row, or JSON record for internal research, client checks, or lead cleanup.

Best first customers

Local-service operators, small agencies, internal support teams, SEO reviewers, and business researchers that need fast contact discovery from public websites.

Hard boundary

This tool should not bypass logins, scrape protected systems, defeat CAPTCHAs, ignore site limits, or send automated outreach. It is a review-first extractor, not a spam engine.

Reusable value

The same parser can support SEO checks, support-link audits, admin intake, product-directory QA, and website cleanup workflows across the Maxxed portfolio.

Candidate fields

Export profile

The exported record should stay practical and auditable.

Core identity

Business name, site title, canonical URL, resolved domain, and review status.

Contact channels

Email addresses, phone numbers, contact form URLs, booking links, and support links.

Presence

Social profiles, address candidates, privacy URL, terms URL, and source snippets.
