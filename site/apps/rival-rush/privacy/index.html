Rival Rush Privacy Policy

Rival Rush is a local party game. The audited internal-test source uses local player profiles, local gameplay metrics, a no-operation production analytics provider, an unavailable Google Play Games sign-in stub, and a no-operation advertising provider unless an advertising build flag and provider are deliberately enabled. Advertising, analytics, Google Play Games, and remote services must be reviewed and disclosed before any build enabling them is released.

Data processed

  • Player display names, avatar selections, settings, and local profile identifiers
  • Local wins, losses, scores, coins, experience, unlocks, challenges, selected game modes, and progress
  • Local gameplay counters such as game mode opened, match started, match completed, round results, and reset events
  • Diagnostic logs in development or internal-test builds, which may include scene names, validation messages, device details, and error traces
  • Advertising ID only if a future ad-enabled production build activates a disclosed advertising provider
  • Support or beta details only if the user chooses to email them, such as tester account, app version, device model, screenshots, scene names, or bug notes

Permissions and purpose

  • Internet: Reserved for Google Play delivery and any production service explicitly enabled in a future build.
  • Advertising ID: Used only if an ad-enabled build activates a disclosed advertising provider. The current no-operation provider does not request an ad.
  • Notifications: May be used for game reminders only if the user grants permission and the feature is enabled.

Storage and security

Player identity, profile, progress, settings, rewards, and gameplay metrics are stored locally through the game's save system. The current Google Play Games provider is a non-functional stub and does not create a remote game account in the documented internal-test build.

Collection and sharing

The current production analytics path is no-operation and does not send gameplay events to Maxxed Technical Systems. The current advertising path is also no-operation unless an ad SDK is deliberately enabled for a later build. If a later build enables ads, analytics, cloud saves, leaderboards, purchases, or Google Play Games sign-in, the policy and Play Data safety declaration must be updated before release.

Retention

Local profile and gameplay data remain until reset in the app where available, Android app storage is cleared, or the game is uninstalled. Beta/support emails are retained only as long as reasonably needed for support, test administration, abuse prevention, operational records, or legal obligations.

Deletion

Clear local profile or progress controls where offered, or use Android clear-storage or uninstall to remove all app-private game data. Email privacy@techmaxxed.com to request deletion of beta/support records sent to Maxxed Technical Systems.

Third-party services

Google Play processes installation, account, purchase, security, crash, and platform diagnostics under Google's terms. If Unity Ads, another ad provider, Google Play Games, Firebase, remote analytics, or cloud services are enabled later, this policy and the Play Data safety declaration must be updated before that build is released.

Children

Rival Rush is intended for a general audience. The final age targeting, advertising configuration, family-policy posture, and store declarations must be reviewed together before public release.

Contact

Questions may be sent to privacy@techmaxxed.com.
