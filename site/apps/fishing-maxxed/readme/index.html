Fishing Maxxed README

An offline-first catch journal with guided photos, known-reference length measurement, uncertainty, manual species confirmation, and privacy-minded exports.

Purpose

Photograph a catch, measure it against a real reference, adjust the endpoints, confirm the species yourself, and keep a local record without exposing exact coordinates by default.

Current Status

Beta testers needed. Functional source is present. Build, signing, known-object measurement tests, device evidence, and regulation-source verification remain in progress.

Pre-release Testing

Even if Fishing Maxxed is still in development, testers can request access. A clear request helps us prioritize the next launch pass, prepare a tester build quickly when feasible, and collect real-device feedback before public release.

Core Workflow

  1. Reference-based measurementMeasure length from a known reference with draggable correction handles and visible uncertainty.
  2. User-confirmed recordsSpecies identification remains an explicit user decision rather than an automated claim.
  3. Private local journalStore catch records on the current device and redact exact coordinates from standard exports.
  4. Truthful rule handlingRegulation fixtures can exercise the evaluator but cannot authorize keeping a fish.

Privacy

Catch records stay private on the device, and default exports redact exact coordinates. The full app-specific policy is available at Fishing Maxxed Privacy Policy.

Known Limitation

Users must check current official fishing regulations. The app cannot authorize a catch as legal to keep.

Weekly Review

This README should stay in sync with the app source, privacy policy, support flow, testing status, and latest GitHub PR work. Review it at least once per week and update stale claims before launch decisions.
